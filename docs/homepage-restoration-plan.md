# PMagic AI 独立官网高保真还原规划

## 目标

将一张完整参考图还原为独立产品官网。页面使用 HTML/CSS/JS 实现，桌面端贴近参考图，同时具备响应式能力。实现不依赖固定长图，不把整张设计稿作为网页背景。

## 最高优先级参考

- 唯一视觉事实源：`reference/source/pmagic-homepage-reference.png`
- 工作切片清单：`reference/sections/manifest.md`
- 原图尺寸：`2876 x 8752`
- 建议桌面实现基准：按约 2x 设计稿理解，桌面内容宽度约 `1200px - 1280px`
- 页面性质：建工领域 AI 产品官网
- 视觉关键词：白底科技感、蓝紫渐变、玻璃拟物 3D、柔和发光、圆角卡片、轻量阴影

## 还原原则

1. 桌面端优先做到高保真，包括区块顺序、层级、间距、卡片形态、主色、阴影和主视觉比例。
2. 响应式需要重新组织布局，保证移动端文字可读、图片清晰、卡片不拥挤。
3. 所有可编辑文案都使用 HTML 原生文本，不嵌入截图文字。
4. 图标、按钮、卡片、标签、线条、背景渐变尽量用代码或 SVG 还原。
5. 中心 3D 标志、模块配图、楼盘案例图、白皮书书本、底部城市光效等使用独立高清图片资产。
6. 后续生成图片时只生成独立配图，保持原图风格，不重新设计页面。

## 实施流程

### 阶段 1：固定事实源

- 保存完整参考图。
- 保存分区切片。
- 记录尺寸、命名、模块顺序和还原重点。

### 阶段 2：语义拆解

- 把长图拆成页面模块。
- 识别哪些内容是文案、哪些是代码图形、哪些是独立图片资产。
- 将每个模块转为可实现的 HTML 结构。

### 阶段 3：静态页面搭建

- 创建 `index.html`、`styles.css`、`script.js`。
- 先实现布局、文案、卡片和按钮。
- 使用临时或独立资产占位，不使用长图背景。

### 阶段 4：视觉资产替换

- 替换 Logo、3D 主视觉、模块图、案例图、白皮书图、城市光效。
- 统一蓝紫渐变、阴影、卡片半径、图片比例。
- 建立差异清单，逐项修正。

### 阶段 5：响应式适配

- 分别检查桌面、平板、手机。
- 移动端按内容重排。
- 确认所有文本可读、按钮可点、图片不遮挡、卡片不溢出。

### 阶段 6：发布验证

- 清理无用素材。
- 检查图片格式、视频 fallback、缓存参数和 GitHub Pages 实际结果。
- 用浏览器截图和参考图对照。

## 验收标准

1. 桌面端打开页面，整体观感与参考图高度一致。
2. 页面不是固定宽度长图，布局随视口变化。
3. 所有正文、标题、按钮、导航均为 HTML 文本。
4. 核心配图为高清独立资产，不使用低清截图放大。
5. 移动端无横向滚动、无文字重叠、无按钮挤压。
6. 首屏品牌、标题、主 CTA、主视觉和下一屏预告都清晰可见。
7. 每次大改后都用浏览器截图和参考图做对照。
